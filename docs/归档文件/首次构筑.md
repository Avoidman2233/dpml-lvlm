# 首次构筑指南 — ATPrompt 元任务微调框架

> **项目全称**：基于元任务微调与属性辅助推理的 LVLM 小样本学习框架  
> **英文**：Meta-Task Fine-Tuning with Attribute-Assisted Reasoning for LVLM Few-Shot Learning  
> **最后更新**：2026-05-09

---

## 1. 项目背景

ATPrompt 通过在 CLIP 的文本 Prompt 中注入 GPT-4o 生成的属性描述词（shape/color/material/function/size 等 31 种组合），提升小样本图像分类性能。现有方法（CoOp/CoCoOp/MaPLe/DePT）在 base 类上做标准监督训练，再在 new 类上 zero-shot 评估。

本次构筑的目标是在此基础上**新增 MAML 风格的元预训练阶段**：在 base 类数据上构造 N-way K-shot episode（support + query），通过 inner/outer loop 训练 prompt_learner 参数获得快速适应能力。元预训练后的 checkpoint 可被标准 trainer 加载，进行常规 fine-tune 和评估。

### 技术路线

```
Phase 1: Meta-Pretrain (新增)
  └─ EpisodicSampler 构造 episode
  └─ MetaPretrainer: MAML inner/outer loop
  └─ 完整二阶梯度（retain_graph + create_graph）

Phase 2: Standard Fine-tune (现有)
  └─ CoOp_ATP / CoCoOp_ATP / MaPLe_ATP / DePT_ATP
  └─ 加载 meta checkpoint → 标准 fine-tune

Phase 3: Evaluation (现有)
  └─ Base-to-new 评估协议
  └─ Base Acc / New Acc / H-Mean
```

### 可移植性设计

- 单一 `MetaPretrainer` 通过 config 驱动 factory pattern 支持 **8 种 prompt 方法**：
  - CoOp / CoOp_ATP / CoCoOp / CoCoOp_ATP
  - MaPLe / MaPLe_ATP / DePT / DePT_ATP
- 跨数据集 / 单数据集两种 meta-training 模式可切换
- Meta checkpoint 格式与标准 trainer **向后兼容**

## 4. 核心代码说明

### 4.1 EpisodicSampler（数据层）

**文件**：`Dassl.pytorch/dassl/data/episodic_sampler.py`

```python
class EpisodicSampler(Sampler):
    """N-way K-shot episode 构造"""
    
    def __init__(self, data_source, n_way, k_support, k_query, n_episodes):
        # 按 label 分组索引
        # 验证：n_way ≤ num_classes, k_support+k_query ≤ min_samples_per_class
    
    def __iter__(self):
        for _ in range(n_episodes):
            selected_labels = random.sample(self.labels, n_way)      # 随机选 N 类
            for label in selected_labels:
                sampled = random.sample(indices, k_support + k_query) # 每类采样
                yield support_indices, query_indices                  # 返回元组
```

### 4.2 MetaPretrainer（训练层）

**文件**：`trainers/meta_pretrainer.py` — 核心类，包含 9 个关键方法：

| 方法 | 行号 | 功能 |
|------|------|------|
| `check_cfg()` | 56 | 验证 meta 配置完整性 |
| `build_data_loader()` | 88 | 构建 EpisodicSampler + DataManager |
| `build_model()` | 121 | Factory：动态导入 8 种 CustomCLIP |
| `run_epoch()` | 194 | Episode-based MAML 训练循环 |
| `_run_episode()` | 227 | 单 episode：inner loop + outer loop |
| `maml_inner_loop()` | 273 | 克隆参数 → K步 SGD → 二阶梯度 |
| `use_fast_weights()` | 331 | 上下文管理器：临时替换参数 |
| `after_epoch()` | 349 | 保存 checkpoint |
| `load_meta_checkpoint()` | 374 | 加载到标准 trainer |

**MAML 核心流程**：

```
每个 episode:
  ┌─ support set → maml_inner_loop()
  │    ├─ clone prompt_learner 参数 → fast_weights
  │    ├─ for step in range(inner_steps):
  │    │    ├─ 加载 fast_weights 到模型
  │    │    ├─ forward → CE loss
  │    │    ├─ torch.autograd.grad(loss, fast_weights, create_graph=True)
  │    │    └─ fast_weights[i] -= inner_lr * grad   (保留计算图)
  │    └─ return fast_weights
  │
  ├─ query set → use_fast_weights()
  │    └─ forward → meta_loss
  │
  └─ meta_loss.backward() → optim.step()   (二阶梯度更新)
```

### 4.3 Config 系统

**META 命名空间**（在 `train.py` 的 `extend_cfg()` 中定义）：

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `ENABLED` | bool | False | 启用 meta 模式 |
| `METHOD` | str | CoOp_ATP | prompt 学习方法 |
| `N_WAY` | int | 5 | 每 episode 的类别数 |
| `K_SUPPORT` | int | 1 | 每类 support 样本数 |
| `K_QUERY` | int | 3 | 每类 query 样本数 |
| `INNER_LR` | float | 0.01 | 内循环学习率 |
| `INNER_STEPS` | int | 5 | 内循环 SGD 步数 |
| `N_EPISODES` | int | 100 | 每 epoch 的 episode 数 |
| `SECOND_ORDER` | bool | True | 使用二阶梯度 |
| `CROSS_DATASET` | bool | False | 跨数据集模式 |
| `DATASETS` | list | [] | 跨数据集列表 |

## 8. 常见问题

### Q1: Import 报错 "No module named 'torch'"

确保 PyTorch 已安装且 CUDA 版本匹配：
```bash
python -c "import torch; print(torch.__version__)"
```

### Q2: 元训练时 loss 出现 NaN

常见原因：
- 学习率过高 → 降低 `TRAINER.META.INNER_LR`（如 0.001）
- Episode 太小 → 增大 `K_SUPPORT` 或 `K_QUERY`
- 梯度爆炸 → 先试 `SECOND_ORDER False`（FOMAML）

### Q3: 跨数据集元训练如何构造数据？

设置 `TRAINER.META.CROSS_DATASET True` 和 `TRAINER.META.DATASETS ["imagenet","caltech101","..."]`。EpisodicSampler 会从所有数据集的 base 类中混合采样 episode。

### Q4: Meta checkpoint 如何加载到标准 trainer？

```python
from trainers.meta_pretrainer import MetaPretrainer
from trainers.coop_atp import CoOp_ATP

# 创建标准 trainer（build_model 会自动构建）
standard_trainer = CoOp_ATP(cfg)

# 加载 meta checkpoint
MetaPretrainer.load_meta_checkpoint(
    standard_trainer,
    "output/meta_pretrain/eurosat/.../prompt_learner/model-best.pth.tar"
)

# 然后正常 fine-tune
standard_trainer.train()
```

### Q5: V100 (16GB) 能跑吗？

可以。建议调整：
```bash
# 减少 batch 内样本数
TRAINER.META.N_WAY 3 \
TRAINER.META.K_QUERY 2 \
TRAINER.META.N_EPISODES 50 \
OPTIM.MAX_EPOCH 10
```
