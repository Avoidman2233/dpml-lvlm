# 最终实验报告：元任务微调与属性辅助推理的 LVLM 小样本学习

## 1. 技术路线

### 1.1 整体架构

```
CLIP ViT-B/16 (frozen backbone)
    │
    ├── Image Encoder → image features
    │
    └── PromptLearner (trainable)
            ├── ctx vectors (learnable context tokens, N_CTX=2)
            ├── ctx_att vectors (learnable attribute tokens)
            └── tokenized [prefix] [attr_word] [class_name]
                    │
                    ▼
            Text Encoder → text features
                    │
                    ▼
            cosine similarity → logits → CE loss
```

### 1.2 ATPrompt — 属性辅助推理

每数据集使用论文指定属性词：

| 数据集 | ATT_NUM | 属性词 | N_CTX |
|--------|---------|--------|-------|
| EuroSAT | 1 | habitat | 2 |
| DTD | 3 | pattern, color, design | 4 |
| Oxford Pets | 2 | playfulness, energy | 2 |
| Oxford Flowers | 3 | color, habitat, growth | 2 |
| FGVC Aircraft | 2 | design, range | 2 |
| UCF101 | 1 | precision | 2 |
| Food101 | 2 | flavor, preparation | 2 |
| Stanford Cars | 1 | luxury | 2 |

### 1.3 MAML 元任务微调

**Inner Loop（快速适应）**:
```
fast_weights = clone(prompt_learner)
for step in 1..K:
    loss = CE(model(support), labels)
    grad = ∂loss/∂fast_weights
    fast_weights -= inner_lr × grad
```

**Outer Loop（元更新）**:
```
meta_loss = CE(model_fast(query), query_labels)
meta_loss.backward() → update prompt_learner (二阶)
```

### 1.4 实验管线

```
Phase 1: Meta-Pretrain (MetaPretrainer, 20 epochs)
    N-way × 1-support × 10-query episodes
    200 episodes/epoch, inner_steps=2, second_order=True

Phase 2: Fine-tune (CoOp_ATP, 100 epochs)
    加载 Phase 1 checkpoint, 标准交叉熵训练

Phase 3: Evaluation
    Base-to-new: new class zero-shot accuracy
```

### 1.5 统一超参数

| 参数 | 值 |
|------|-----|
| Backbone | ViT-B/16 CLIP (frozen) |
| Shot | 16 |
| N_CTX | 2 (DTD=4) |
| FT Epochs | 100 |
| Meta Epochs | 20 |
| N_WAY | 3-40 (per dataset) |
| K_SUPPORT | 1 |
| K_QUERY | 10 |
| INNER_STEPS | 2 |
| SECOND_ORDER | True |
| Seeds | 1, 2, 3 |

## 3. 分析与讨论

### 3.1 核心发现

1. **MAML meta-pretraining 在 100 epoch FT 下效果有限**：仅 Oxford Flowers (+1.2%) 和 Stanford Cars (+3.1%) 正向，其余 6 个数据集负向或持平。

2. **最佳场景是细粒度大类别数据集**：Stanford Cars (196 类) 三种子一致正向，每 seed 提升 2.8-3.9%。

3. **100 epoch FT 覆盖 meta 初始化优势**：MAML 的核心价值是快速适应，但 100 epoch 标准训练使得两种初始化（随机 vs meta）收敛到同一局部极小值。

4. **属性词对齐显著提升 baseline**：修复属性词后，EuroSAT 从 51.5% → 62.2%（超过论文）。

### 3.2 局限

- MAML 在小数据集上（<50 base 类）episode 多样性不足
- 二阶 MAML 内存开销大，限制 N_WAY 上限
- FT 100 epoch 与 MAML "快速适应"初衷矛盾
- 未尝试跨数据集 meta-training（论文目标能力）

### 3.3 改进方向

1. **跨数据集 meta-training**：ImageNet 500 base 类 → 丰富 episode 分布 → 下游数据集
2. **减少 FT epoch**：10-30 epoch 替代 100，保留 meta 初始化优势
3. **ANIL / ProtoMAML**：替代完整二阶 MAML，更高效
4. **ViT-L/14 backbone**：提升 baseline，放大 meta 增益
