# DL-MPT: 基于 Episodic Regularization 的元任务增强 Prompt 学习方法

## 一、问题定义

小样本视觉-语言模型（LVLM）分类任务中，如何有效引入元任务学习方法进行改进。

具体而言，在 CLIP ViT-B/16 (frozen backbone) 的 Prompt Learning 框架下，以 CoOp+ATP（属性辅助推理的可学习 prompt）为基线，探索元任务训练机制能否进一步提升小样本泛化性能。

---

## 二、基线方法: CoOp+ATP

CLIP 的 prompt 模板 `a photo of a {classname}` 被替换为可学习的连续向量:

```
prompt = [ctx_1, ctx_2, attr_1, ..., ctx_k, {classname}_tokens]
```

其中 `ctx` 是共享的可学习上下文向量，`attr` 是属性增强向量（如 "luxury", "color", "texture"），`classname` 是类别名称的词嵌入。训练时仅优化 prompt 参数（约 4K），CLIP 的 image encoder 和 text encoder 完全冻结。

训练目标为标准交叉熵:
```
L_base = CE(cos_sim(image_features, text_features), label)
```

---

## 三、DL-MPT 方法

### 3.1 核心架构

DL-MPT (Dual-loop Meta Prompt Tuning) 在 CoOp+ATP 的基础上引入第二条训练路径——元任务原型分类:

```
L_total = L_base + λ · L_meta
```

**Path 1 (L_base)**: 标准 CoOp+ATP 分类。全 batch 图像通过 image encoder，prompt 通过 text encoder 生成 98 类的文本特征，计算 cosine similarity 后用交叉熵优化。

**Path 2 (L_meta)**: 元任务原型分类。每个 batch 从 base 类中随机采样一个 N-way K-shot episode（N=20 类，K=3 张 support，K=10 张 query），计算类别原型并分类 query:

```
vis_proto  = mean(image_encoder(support))     // 视觉原型
text_proto = text_encoder(prompt)[class]       // 文本原型
prototype  = (vis_proto + text_proto) / 2      // 多模态融合
L_meta     = CE(cos_sim(query_features, prototype), query_label)
```

**λ 调度**: 三阶段训练共 25 epochs:
- Warmup (ep 1-5): λ = 0.00，仅 L_base，prompt 稳定初始化
- Joint   (ep 6-20): λ = 0.20，联合优化
- Refine  (ep 21-25): λ = 0.50，强调泛化

### 3.2 实现细节

- 所有训练流程固化为 `.sh` 脚本，确保可复现
- 训练输出重定向使用 `> log 2>&1`，禁止 `| tee`
- 固定格式进度日志，记录 L_base / L_meta / acc_meta / λ / lr
- 评估协议分为 Protocol A (Base-to-New) 和 Protocol B (episodic)

### 3.3 命名规范

基线方法是 CoOp+ATP 和 CoCoOp+ATP，DL-MPT 注入其中:

| 正确命名 | 含义 |
|---------|------|
| `DL-MPT(CoOp+ATP)` | 基于 CoOp+ATP 的双环训练（含属性词） |
| `DL-MPT(CoCoOp+ATP)` | 基于 CoCoOp+ATP 的双环训练（含属性词） |

## 五、L_meta 作用机制分析

### 5.1 核心发现: L_meta 是 Episodic Regularization，而非 Meta-Learning

对训练过程中的 L_meta 观测显示:

```
Epoch 1  (λ=0):  L_meta = 2.879, acc_meta = 75%
Epoch 25 (λ=0.5): L_meta = 2.865, acc_meta = 73%
```

L_meta 在整个训练过程中几乎没有下降——原型网络的分类能力本身并未改善。这表明 L_meta 的有效性不来自"学会了更好的原型"，而来自以下机制:

**Episodic Regularization**: 每个 batch 的 meta path 随机采样不同的 20 类 episode。prompt_learner 每个 batch 收到的梯度方向随 episode 变化:
```
Batch 1: 梯度方向由类 {3,17,22,41,...} 的 episode 决定
Batch 2: 梯度方向由类 {1,8,14,29,...} 的 episode 决定
Batch 3: 梯度方向由类 {5,11,25,38,...} 的 episode 决定
```

这 100 个不同方向的梯度在 prompt_learner 的 4K 参数空间上叠加，形成**隐式随机正则化**——防止 prompt 过拟合到固定的 base 类分布，从而提升 Novel 泛化。

### 5.2 关键证据: 所有 Structural Enhancement 全部失败

我们系统性地尝试了 8 种方法在 meta path 中引入可学习的适配器、对齐器或优化器改进:

| 尝试方向 | 变体数 | 机制 | 结果 |
|---------|:---:|------|------|
| Patch 级 adapter | 2 | MLP / Cross-Attention 改造视觉原型 | Novel 均不退，Base 下降 |
| Proto 级 refiner | 2 | 残差 MLP / +alignment loss | Base 提升，Novel 不动 |
| Token 级 aligner | 2 | Cross-Attention + InfoNCE / 属性门控 | 全部失败 |
| 真 Meta-Learning | 2 | Reptile inner-loop / + L2 约束 | 退化至 67% / 打平 |

**结论**: prompt_learner 的优化完全被 L_base 的交叉熵梯度主导。所有在 meta path 中引入的增补模块或损失函数，其梯度均需穿过冻结的 CLIP encoder 才能到达 prompt_learner，信号衰减至不可用。L_meta 的有效性仅来自梯度方向的随机性（episodic sampling），而非来自原型质量或对齐质量的提升。

## 六、方法论定位

### 6.1 与已有方法的区别

DL-MPT 使用 episodic training + prototype classification 的形式，表面上类似 ProtoNet 或 MAML，但存在关键差异:

- **ProtoNet**: 使用 episodic prototype 训练整个网络，目的是学习更好的特征表示。DL-MPT 仅训练 4K prompt 参数，CLIP encoder 完全冻结。
- **MAML**: 内环 SGD 适应 + 外环二阶优化，目标是学习易于微调的初始化。DL-MPT 无内环适应，本质是单环优化。
- **TIP-Adapter**: 使用 prototype cache 做推理时增强，无训练过程。

**DL-MPT 的独特贡献**: 首次将 episodic prototype training 作为 prompt learning 的**联合正则化项**（而非替代品）。这与将 episodic training 作为主要训练范式的 ProtoNet 有本质区别——DL-MPT 不依赖 episodic 信号进行主要学习，仅利用其随机性提供泛化约束。

### 6.2 实验结论

在 8 个数据集上的 3-seed 实验中，DL-MPT 稳定地以 25 epochs 的小训练量超越了 100 epochs 的标准 CoOp+ATP。消融实验证明:

1. episodic sampling 的随机梯度方向是泛化增益的唯一来源
2. 任何试图在 meta path 中引入学习信号的结构性增强均无法进一步提升
3. 属性辅助推理（ATP）对 prompt 学习的贡献独立于 episodic 信号
